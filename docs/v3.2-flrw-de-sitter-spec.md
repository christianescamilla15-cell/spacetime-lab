# v3.2 deferred — FLRW + de Sitter cosmologies (spec)

**Status:** deferred from v3.2.  Cosmologies are a different beast
than the BH metrics (no horizon, no ISCO, but scale factor a(t) and a
matter-content equation of state).  Honest scope: 1 week per metric,
plus shared cosmology UI work.

This document is the spec for when we eventually pick it up.

## Why "different beast"

The whole `metrics/` subpackage so far assumes static or stationary
backgrounds: a fixed metric tensor, a notion of "horizon as a 2-sphere
in space", "ISCO as a circular orbit", "Hawking T as a constant".
None of these survive in time-dependent cosmological backgrounds.

For FLRW we need:
- **Scale factor a(t)** as a free function (typically chosen by an
  equation of state w: pressure / energy density)
- **Friedmann equation** as the dynamical content:
  `(ȧ/a)² = (8πG/3) ρ − k/a²`
- **Particle / event / cosmological horizons** as integrals
  `∫₀^t dt'/a(t')`, *not* the algebraic root of g_tt = 0
- **Matter content choice** (radiation, matter, Λ, mixtures)
- **Spatial curvature parameter k ∈ {-1, 0, +1}**
- **Hubble parameter H(t) = ȧ/a** as the natural observable

For de Sitter:
- Pure-Λ vacuum, exact closed-form a(t) = exp(H t) with H = √(Λ/3)
- Cosmological event horizon at distance 1/H from any observer
- Tunes to FLRW with w = -1, k = 0

These are physically simple, software-architecturally different.

## Proposed architecture

### New base class

```python
class CosmologicalMetric(Metric):
    """Time-dependent metric a(t)·g_spatial.

    Subclass requirements:
      .scale_factor(t) -> float
      .hubble_parameter(t) -> float          # = a'/a
      .deceleration_parameter(t) -> float    # = -a''·a / (a')²
      .equation_of_state -> float | None     # constant w; None for mixed
    """

    def particle_horizon(self, t: float) -> float: ...
    def event_horizon(self, t: float) -> float: ...
    def comoving_distance(self, z: float) -> float: ...
```

Cleanly separates from the static `Metric` base — they share only the
`metric_tensor` interface, not the BH-specific helpers (`outer_horizon`,
`isco`, `hawking_temperature`).

### New modules

```
spacetime_lab/metrics/cosmology/
    __init__.py
    base.py                      # CosmologicalMetric ABC
    de_sitter.py                 # exact exponential
    flrw.py                      # general k, w, mixed components
    friedmann_solver.py          # numerical ODE for general matter content
```

### Backend endpoints

```
GET /api/cosmology/de-sitter?H=&t=
    → {scale_factor, hubble, event_horizon}

GET /api/cosmology/flrw?w=&k=&Omega_m=&Omega_r=&Omega_L=&t=
    → {scale_factor, hubble, age, particle_horizon, event_horizon, deceleration}

GET /api/cosmology/flrw/evolution?...&t_max=&n_samples=
    → {t: [...], a: [...], H: [...], q: [...]}
```

### Frontend page

`/cosmology` (one page covering both, with a subnav: de Sitter / FLRW)

Plot ideas:
- a(t) curve with H₀ and Λ markers
- Hubble diagram (z vs distance)
- Particle horizon vs time (the famously expanding "observable
  universe")
- ΛCDM matter-radiation-Λ epochs
