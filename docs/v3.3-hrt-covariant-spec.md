# v3.3 deferred — HRT covariant (Hubeny-Rangamani-Takayanagi) spec

**Status:** deferred from v3.3.  Multi-day theory + numerics work.

The current `holography/minimal_surfaces.py` only handles **static**
RT surfaces — geodesics on a constant-time slice in Euclidean-signature
hyperbolic geometry.  HRT generalises to **time-dependent** backgrounds
where the minimal surface is *spacelike but extremal* in Lorentzian
signature, requiring a numerical extremal-surface finder that handles
both space-like and time-like coordinate variations.

## Why it's multi-day

1. **Lorentzian geometry**: the area functional changes sign meaning
   — extremality replaces minimisation.  Standard `scipy.optimize`
   minimizers don't apply directly.
2. **Embedding parametrisation**: a 2D surface in 4D (or higher) needs
   a careful parametric form (e.g., depth profile z(x, t)) plus
   parameter-domain choice.
3. **Maximin formulation** (Wall 2014): HRT extremal surface = maximum
   over time slicings of minimum-area Cauchy surface.  This is the
   numerically-tractable reformulation but it requires nested
   optimisation.
4. **Validation**: results must agree with the static RT in the
   stationary-background limit (e.g., AdS-Schwarzschild Vaidya ringing
   into equilibrium — at late times HRT must match static RT in the
   final BH background).

## Existing assets we'd build on

## Risk register

- **R1**: Numerical extremization is fragile — start with simple bracket
  searches in 1-parameter problems, only generalize to 2D after that
  works
- **R2**: Lorentzian extremal surfaces can develop caustics — flag and
  refuse to extrapolate beyond them
- **R3**: Vaidya bounce point (where the null shell hits r = 0) may
  numerically misbehave; need a regulator

## Connection to existing work

The current Page-curve modules (`holography/island.py`,
`holography/evaporating.py`) used the **identified** HM saddle as a
direct closed-form; HRT covariant would replace that with a
**computed** extremal surface in a dynamical background, closing the
last conceptual gap between the static and dynamical sides of the
holographic information-paradox machinery.

Connection to v2.0 QES finder: HRT is the geometric counterpart to
QES.  Once both are dynamical, the four-way comparison (static RT,
static QES, dynamical HRT, dynamical QES) becomes possible — the
ultimate cross-check of the formalism.
